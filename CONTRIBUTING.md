Thank you for considering contributing to this project! Contributions that improve clarity, portability, tests, or documentation are welcome.

How to contribute
- Fork the repository and create a feature branch.
- Make small, focused commits with clear messages.
- Update `README.md` or add documentation for any new features.
- Open a Pull Request describing your change.

Coding conventions
- Prefer readable, portable C (C11 where reasonable).
- Keep functions small and focused.

Reporting issues
- Open an issue describing the problem and steps to reproduce.

License compatibility
- Contributions are accepted under the MIT License.
